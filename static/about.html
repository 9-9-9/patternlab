Pattern Lab

Creating pattern libraries with atomic design

Pattern Lab is a collection of tools to help create and maintain robust web design systems.

What Pattern Lab Isn't

  • A UI framework—Pattern Lab isn't Bootstrap. It's not Foundation. It's not flat. It's not skeumorphic. Pattern Lab provides the plumbing for creating design systems. How it looks is entirely up to you.
  • Language-dependent—Like Sass? Cool, go nuts with it. Like Less instead? That's cool too. Don't like any preprocessors at all?
  • Incredibly Rigid—Pattern Lab is meant to fit in with the way you're used to doing things. There's really no right way to do this

Why

Tiny Bootstraps, for Every Client

Dave Rupert

This style guide keeps track of every module going onto the site, creates a shared vocabulary between everyone involved in the project, makes testing easier, establishes a better workflow, and builds a canonical reference to keep coming back to.

Who

Pattern Lab benefits everyone involved in a web project. Clients and managers can use it as a way of keeping track of everything going on the site and as an educational resource. Visual designers can use it as a way to see how each module works in a responsive environment and also keep track of typefaces and other visual treatments. Developers can use it as a front-end code library and use it to test everything under one roof.

How

The style guide's patterns live in sub-folders of the patterns directory. The index page dynamically pulls in all the patterns into one page. The styles are helped along with Sass, which allows the team to define variables and author more modular styles.

Where

The style guide should initially live in a sandbox environment accessible via a single URL. This ensures that everyone on the team has access to the style guide. Optionally (and ideally) the style guide may be published publicly like Starbucks, The BBC, and The Guardian have done.
